🎯 Project Overview
The AI Table Ordering Assistant is a next-generation, voice-powered restaurant ordering system designed to transform the traditional dining experience into a seamless, intelligent, and premium interaction.

Instead of relying on waitstaff for order-taking, customers interact directly with an AI assistant through a tablet device placed on their table. This assistant uses natural voice interaction combined with touch-based UI, allowing users to explore the menu, place orders, customize dishes, and complete payments effortlessly.

The system is specifically designed for modern cafés and premium dining environments, where speed, accuracy, and experience are equally important.


✨ Core Experience
From the moment a customer sits at the table, the AI assistant becomes their personal ordering companion:

Greets the user with a warm, premium tone
Understands natural language (e.g., “I want something spicy”)
Recommends dishes intelligently
Allows quick add/remove/modify actions
Handles customizations like spice level, ingredients, or extras
Confirms the full order before processing
Suggests relevant add-ons (upselling)
Enables fast and flexible payment options
This creates an experience that feels:

Effortless (no waiting for staff)
Accurate (reduced human error)
Engaging (interactive AI experience)

⚡ System Intelligence
The assistant is not just a voice interface—it is an intelligent system that:

Interprets user intent (ordering, modifying, asking questions)
Maintains conversational context
Adapts to user preferences
Handles real-time availability from the kitchen
Generates structured data for backend processing
It bridges the gap between customer interaction and kitchen operations, ensuring smooth communication without manual dependency.


🏢 Business Impact
For restaurants, this system delivers:

Reduced dependency on waitstaff
Faster table turnover
Increased average order value through smart upselling
Fewer order errors
Streamlined kitchen communication

🌟 Vision
To create a fully AI-driven dining experience where ordering is:

Faster than traditional systems

More engaging than human interaction
The long-term vision is to evolve this system into a complete restaurant intelligence platform, integrating personalization, analytics, and entertainment to redefine how people dine.


This project is not just about ordering food—it is about reimagining the restaurant experience through AI.

🧠 AI Conversation Flow (System Brain)
The AI Conversation Flow defines how the assistant understands, responds, and guides users throughout the ordering journey.

It is built on three layers:

Intent Detection
Conversation Management
Response Generation

⚙️ 1. Core Conversation Loop
This is the main loop that runs continuously:

USER INPUT → Speech-to-Text → Intent Detection → Entity Extraction → State Update → Response → Text-to-Speech


🎯 2. Intent Detection System
The AI must classify every user input into a clear intent.

Primary Intents
1. Add Item
Examples:

“Add 2 pizzas”
“I want paneer tikka”
→ Action:

Identify item
Identify quantity
Add to cart

2. Remove Item
Examples:

“Remove pasta”
“Cancel one coffee”
→ Action:

Find item in cart
Reduce/remove quantity

3. Modify Item
Examples:

“Make it less spicy”
“Add extra cheese”
→ Action:

Update customization of last/selected item

4. Browse / Explore
Examples:

“Show desserts”
“What do you have in drinks?”
→ Action:

Display category items

5. Ask / Query
Examples:

“What is special today?”
“Is this spicy?”
→ Action:

Provide info
Suggest items

6. Confirm Order
Examples:

“Yes, confirm”
“Place the order”
→ Action:

Move to confirmation → kitchen

7. Cancel / Reset
Examples:

“Cancel order”
“Start over”
→ Action:

Clear cart
Restart flow

🧩 3. Entity Extraction
From every sentence, AI extracts:

Item Name → “Paneer Tikka”
Quantity → “2”
Category → “Dessert”
Customization:

Spice level
Ingredients
Add-ons

🧠 4. Conversation State Management
The AI maintains a live session state:

Example State:
Current Cart
Last Added Item
Active Screen (menu/cart/payment)
Pending Question (e.g., spice preference)

Example Flow:
User: “Add pasta”
→ AI: “How spicy would you like it?”
→ (State: waiting_for_spice_level)

User: “Medium”
→ AI completes customization


🔄 5. Conversation Flow Scenarios

🟢 Scenario 1: Simple Order
User: “I want 1 cold coffee”

AI:

Detects intent: Add Item
Extracts:

Item: Cold Coffee
Quantity: 1
Response:
“Cold coffee added to your order”


🟡 Scenario 2: Multi-Step Order
User: “Add paneer tikka”

“How spicy would you like it?”

🔵 Scenario 3: Exploration → Order
User: “Show desserts”

AI:
(Displays desserts)

User: “Add chocolate cake”

“Chocolate cake added to your order”


🔴 Scenario 4: Correction Flow
User: “Add pasta”

AI:
“Did you mean white sauce pasta or red sauce pasta?”

User selects → continues


🟣 Scenario 5: Unavailable Item
User: “Add pizza”

AI:
“Pizza is currently unavailable. Would you like to try garlic bread or pasta instead?”


💬 6. Response Generation Rules
The AI must follow strict communication rules:

Tone
Premium
Friendly
Short

Structure
✅ Good:
“Paneer tikka added. Would you like anything else?”

❌ Bad:
“Your item has been successfully added to the cart. What else would you like to do?”


Rules:
Max 1–2 sentences
No complex words
Always guide next action

🔁 7. Context Awareness
AI must remember:

Last item:
“Make it spicy” → applies to last item
Current flow:
If in payment → don’t suggest menu again

⚠️ 8. Error Handling Flow

Case 1: Unknown Item
User: “Add xyz”

AI:
“I couldn’t find that. Here are similar options.”


Case 2: Low Confidence
AI:
“Did you say Paneer Butter Masala?”

→ Yes / No options


Case 3: Silence
AI:
“Would you like me to show the menu?”

🖥️ Technology Stack & AI Architecture (Free & Practical)
This project is designed to be built using free, local-first, and scalable technologies, ensuring low cost while maintaining performance.


🧱 1. Overall Architecture
The system consists of 4 main layers:

Frontend (Tablet UI)
Backend (Server + Logic)
AI Engine (Local / Free Model)
Speech Layer (Voice Input & Output)

🎨 2. Frontend (User Interface)
Technology:
React.js
Tailwind CSS
Why?
Fast UI development
Easy component system
Perfect for tablet web apps
Role:
Displays menu, cart, screens
Handles touch interaction
Shows AI responses (text + UI)
Integrates voice input/output

⚙️ 3. Backend (Server)
Technology:
Node.js (Express.js)
Role:
Handles API requests
Manages orders & cart
Communicates with AI model
Sends data to kitchen system

🤖 4. AI Engine (FREE & LOCAL)
✅ Recommended Option: Ollama (Local AI)
Model:
llama3:8b
Why this?
100% FREE
Runs locally (no API cost)
Good enough for conversation + intent detection
Works with Node.js easily

🧠 AI Responsibilities
Understand user input
Detect intent (add/remove/modify)
Extract entities (item, quantity, customization)
Generate responses
Return structured JSON

🔁 Flow:
Frontend → Backend → Ollama → Backend → Frontend


🎤 5. Speech-to-Text (Voice Input)
Technology:
Web Speech API (Browser-based)
Why?
FREE
Built into Chrome
No setup needed

How it works:
User taps mic 🎤
Browser listens to voice
Converts speech → text
Sends text to backend

Example:
User says:
“I want 2 cold coffee”

→ Converted to:
"i want 2 cold coffee"

→ Sent to AI


🔊 6. Text-to-Speech (Voice Output)
Technology:
SpeechSynthesis API (Web Speech API)
Why?
FREE
Works directly in browser
No external API needed

How it works:
AI generates response text
Frontend triggers speech synthesis
Tablet speaks the response

Example:
AI Response:
“Cold coffee added to your order”

→ Converted to voice output automatically


🎛️ Voice Settings:
Voice: English (India)
Speed: 0.9 (slightly slower for clarity)
Pitch: 1

🎤 MicButton.jsx (Voice Input)
export default function MicButton() {

const startListening = () => {
const recognition = new window.webkitSpeechRecognition();
recognition.lang = "en-IN";

recognition.onresult = (event) => {
const text = event.results[0][0].transcript;
console.log("User said:", text);

// Send to backend
fetch("http://localhost:5000/ai", {
method: "POST",
headers: {
"Content-Type": "application/json",
},
body: JSON.stringify({ message: text }),
});
};

recognition.start();
};

return (
<button
onClick={startListening}
className="bg-[#7EE0D6] text-black px-6 py-3 rounded-full"
>
🎤 Speak
</button>
);
}

⚙️ 3. Backend Starter Code (Node.js)

📄 server.js
const express = require("express");
const cors = require("cors");
const aiRoute = require("./routes/ai");

const app = express();

app.use(cors());
app.use(express.json());

app.use("/ai", aiRoute);

app.listen(5000, () => {
console.log("Server running on port 5000");
});


📄 routes/ai.js
const express = require("express");
const router = express.Router();
const { handleAI } = require("../controllers/aiController");

router.post("/", handleAI);

module.exports = router;


📄 controllers/aiController.js
const { getAIResponse } = require("../services/ollamaService");

exports.handleAI = async (req, res) => {
const { message } = req.body;

try {
const aiResponse = await getAIResponse(message);
res.json(aiResponse);
} catch (error) {
res.status(500).json({ error: "AI error" });
}
};


🤖 services/ollamaService.js
const axios = require("axios");

exports.getAIResponse = async (message) => {
const response = await axios.post("http://localhost:11434/api/generate", {
model: "llama3",
prompt: `
You are a restaurant AI.
Extract:
- intent
- item
- quantity

User: ${message}

Return JSON only.
`,
stream: false,
});

return response.data.response;
};


🔊 4. Text-to-Speech (Frontend Utility)
📄 utils/speech.js
export function speak(text) {
const speech = new SpeechSynthesisUtterance(text);
speech.lang = "en-IN";
speech.rate = 0.9;
window.speechSynthesis.speak(speech);
}


🔗 5. API Helper
📄 utils/api.js
export async function sendMessage(message) {
const res = await fetch("http://localhost:5000/ai", {
method: "POST",
headers: {
"Content-Type": "application/json",
},
body: JSON.stringify({ message }),
});

return res.json();
}
